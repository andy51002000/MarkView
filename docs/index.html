Privacy & security

Opening a file shouldn't be a leap of faith.

  • HTTPS-only remote imagesPlain http:// image URLs are never fetched. file:// and other schemes are always rejected.
  • Local images stay in the document folderAbsolute paths and ../ traversal that escape the opened file's directory are blocked — symlinks resolved before the check.
  • Quick Look never touches the networkPressing Space on an untrusted file fetches zero remote content; remote images show a placeholder instead.
  • Read-only by designMarkView is a viewer. It never modifies, executes, or phones home about your files.
Install

From zero to default Markdown app in 60 seconds.

Requires macOS 13+ and Swift command line tools (no full Xcode needed).

Clone the repo 
git clone https://github.com/andy51002000/MarkView.git

FAQ

Common Questions

Is MarkView free?

Yes. MarkView is completely free and open-source under the MIT license. No paid tier, no subscription, no in-app purchase, no telemetry. Clone from GitHub and build with the included install script.

Is MarkView a viewer or an editor?

Viewer only. MarkView never modifies your files. Pair it with VS Code, Vim, or any editor you already use for writing.

How does MarkView compare to Typora?

Typora is a paid ($14.99) Markdown editor built on Electron, weighing ~90 MB. MarkView is a free, read-only viewer built with native SwiftUI at ~1.2 MB installed. MarkView also supports Quick Look Finder preview (press Space on any .md file) — Typora does not.

How is MarkView different from Marked 2?

Marked 2 costs $13.99 and is closed-source. MarkView is free and MIT open-source. Both are native macOS Markdown viewers, not editors. Marked 2 has more theming and export options; MarkView has a smaller footprint and a stricter security model.

Why is MarkView only ~1.2 MB when other Markdown apps are hundreds of MB?

Obsidian, Typora, and VS Code bundle an entire Chromium browser engine (Electron) to render Markdown. MarkView uses the text rendering frameworks already on your Mac (SwiftUI), so it only ships the code unique to Markdown viewing.

How do I preview Markdown files in Finder by pressing Space?

Run bash install.sh from the MarkView repo. The installer builds and registers a Quick Look extension, so pressing Space on any .md file in Finder shows rendered Markdown instead of plain text. Quick Look mode never fetches remote content.

What macOS version does MarkView require? Do I need Xcode?

macOS 13 Ventura or later, on both Intel and Apple Silicon. Full Xcode is not required — Swift Command Line Tools are enough (xcode-select --install).

Is MarkView safe for untrusted Markdown files?

MarkView isolates untrusted content: only HTTPS remote images are fetched (plain HTTP and other schemes rejected), local images are sandboxed to the document's directory with ../ traversal blocked, and Quick Look previews make zero network requests. Read-only by design.

Does MarkView handle large Markdown files?

Yes. A 5 MB / 115,000-block document parses in ~0.4 s in the background — the UI never freezes.
