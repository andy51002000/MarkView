MarkView vs Typora

Typora is a polished paid ($14.99) Markdown editor built on Electron (~90 MB). MarkView is a free, read-only viewer built with native SwiftUI (~1 MB installed, 660 KB binary). The difference is intent: Typora is where you write; MarkView is what opens when you double-click a .md file just to read it. MarkView also adds Quick Look previews — press Space on a Markdown file in Finder and see it rendered without launching anything. Typora has no Quick Look support.

MarkView vs Marked 2

Marked 2 ($13.99) is the best-known native Markdown viewer on macOS, and the closest product to MarkView in spirit — both are viewers, not editors. Marked 2 offers deeper theming, export, and writer-analytics features. MarkView's case: it's free, MIT open-source, ~1 MB installed, ships a Quick Look extension, and applies a stricter image-security model (HTTPS-only remote images, directory-sandboxed local images, zero network requests in Quick Look). If you need Marked 2's pro features, buy it. If you need a fast, free reader, MarkView covers it.

MarkView vs Obsidian (for reading only)

Obsidian is a knowledge-base platform — vaults, plugins, graph view, sync. If that's your workflow, keep Obsidian. But if you find yourself opening Obsidian just to read a stray README.md, that's a ~213 MB Electron launch for a task MarkView does in ~1 MB with instant startup. MarkView opens any Markdown file directly — no vault, no indexing, no setup.

The pattern

Reading Markdown doesn't require a browser engine. MarkView renders with the frameworks already installed on every Mac, which is why it's more than two orders of magnitude smaller than Electron-based tools while parsing a 5 MB / 115,000-block document in ~0.4 seconds without freezing the UI.
